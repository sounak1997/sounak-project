How the Notification System Works

1
User Registers POST /api/auth/register — MongoDB saves the user
arrow_downward
2
RabbitMQ Publish publishUserRegistered() sends a message to the user_registered queue
arrow_downward
3
Consumer Processes notificationConsumer.js reads the queue and calls broadcastToAll()
arrow_downward
4
SSE → Angular Signal Browser EventSource receives the event → Angular signal() updates → UI re-renders instantly
